###commit

> 팀플에서 커밋 메시지는 "나중에 누가 봐도 이유를 알 수 있게" 쓰는 게 핵심

작성 원리
[#이슈번호] 작업 설명

좀 더 깔끔하게 하려면:
[#이슈번호] docs: 작업 설명
[#이슈번호] chore: 작업 설명
[#이슈번호] fix: 작업 설명
[#이슈번호] merge: 작업 설명

###Branch

main
└─ 항상 안정 상태

feature/<이슈번호>-<작업명>
└─ 작업용 브랜치

1. main 에 직접 push 금지
2. 모든 작업은 feature branch 에서
3. main 반영은 PR 통해서만
4. PR 머지 전 최소 1명 리뷰
